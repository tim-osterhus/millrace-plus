## Public Validation And CI

The public command is the six-file standalone pytest selection in
`docs/public-validation.md`, followed by `uv build`, wheel inspection,
`ruff check src tests`, and `git diff --check`. The checked-in workflow is
`.github/workflows/public-ci.yml` and runs for pull requests and pushes to
`main`.

Public validation requires no sibling runtime checkout, legacy asset checkout,
or `PYTHONPATH`. Internal conformance still has explicit opt-in dependencies on
the pinned runtime source and legacy donor evidence through
`MILLRACE_PLUS_RUN_INTERNAL_CONFORMANCE`, `MILLRACE_RUNTIME_SOURCE`, and
`MILLRACE_LEGACY_ASSET_ROOT`. Those paths are internal-only evidence, are not
hidden public CI requirements, and need no release waiver.

The workflow file and local command prove the checked-in CI boundary. They do
not treat an unobserved future run as evidence. Exact successful post-push CI
for the package handoff commit is mandatory and is recorded in the external
PLUS-0003.9 implementation review, where it can be named without changing the
commit under test. Publication or cutover must consume that exact-SHA result,
not only the earlier successful input-checkpoint run.

`docs/e2e-live-smoke.md`. These files remain source-local development evidence,
not runtime or package surface.

## Unsupported Scope

`millrace-plus` provides no plugin, marketplace, provider, MCP, credential,
purchase, payment, remote-install, or native runner behavior. It supplies
non-executable package data only. Current live runner evidence is Codex-only;
non-Codex provider/native runner support is unavailable. Millrace OS,
`millrace-web`, and Millforge are not v0.22 package contents or dependencies.
